——————>  表示“有一个”的关系。如持有一个引用类型的变量。


---------------------

设计原则：

	找出应用中可能需要变化之处，把他们独立出来，不要和那些不需要变化的代码混在一起。
	针对接口/超类型（Animal a = new Dog()）编程，而不是针对实现（Dog d = new Dog()）编程;
	多用组合，少用继承。
	“有一个”可能比“是一个”更好。
	为了交互对象之间的松耦合设计而努力。
	对扩展开放，对修改关闭。
	
	
	
	
---------------------------------

“有一个”的关系非常有趣，每个鸭子都有一个飞的行为和叫的行为，好将飞和叫委托给他们处理，当你将两个类结合起来使用，这就
是组合(composition)，组合和“继承”的区别是：鸭子的行为不是继承来的，而是和适当的 “行为对象” 组合来的。这是很重要的技巧。
